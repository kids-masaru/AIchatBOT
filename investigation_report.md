# ログ分析と改善計画レポート

頂いたログを基に、現在発生している3つの問題について原因と対策を整理しました。

## 1. 朝のリマインダーが2回届く問題

### 📊 ログからの分析
ログを確認しましたが、**アプリ内部では1回しか送信処理が行われていない**ように見えます。（`Push sent` のログが、それぞれの日付で1回ずつセットになっています）

しかし、実際に2回届いているということは、以下の可能性があります：
1. **プロセス重複**: 前回の修正（workers=1）が正しく反映されていない、または古いコンテナがまだ生きている。
2. **外部トリガーの重複**: アプリ内のスケジューラー(`BackgroundScheduler`)とは別に、VercelのCronなどが `/cron` エンドポイントを叩いている。

### 🛠 対策案
**「二重送信防止ロック」の実装**を推奨します。
- 送信前に「今日、このユーザーに、このリマインダーを送ったか？」をデータベース（Google Sheetsまたはメモリ）で確認する処理を追加します。
- これにより、プロセスが複数あっても、外部から叩かれても、物理的に2回目の送信をブロックできます。
- 具体的には、最後の送信時刻を記録し、一定時間内（例: 20時間以内）の再送信を防ぎます。

## 2. 「昨日の予定」が届く問題 (日付のズレ)

### 📊 ログからの分析
```
[DEBUG] Executing tool: list_calendar_events
Executing: list_calendar_events({'time_max': ... 'time_min': '2026-01-15T00:00:00'})
```
このログが出ている時間が重要です。サーバーはUTC（協定世界時）で動いており、日本時間(JST)より9時間遅れています。
- JST 朝7時は、UTC 前日22時です。
- コード内で単純に `datetime.now()` (UTC) を使うと、日付が「昨日」になってしまいます。

### 🛠 対策案
カレンダー取得ツール(`list_calendar_events`)やAIへのプロンプトで、**必ずJST（日本時間）に変換した日付**を使用するように修正します。
- 「現在は日本時間で〇月〇日です」とAIに強く認識させます。
- `datetime.now()` を呼ぶ全ての箇所でタイムゾーンを考慮します。

## 3. プロファイル（記憶）が保存されない問題

### 📊 ログからの分析
明確なエラーが出ています。
```
Error saving profile: (400)
... "Dense vectors must contain at least one non-zero value..."
```
プロファイルを保存する際、検索用ではないため「全て0」のベクトルデータを入れていましたが、Pineconeの仕様で「全て0のベクトル」が禁止されました。これが原因で保存に失敗し、記憶が更新されていません。

### 🛠 対策案
- 保存時のダミーベクトルを、全て0ではなく `[0.1] * DIMENSION` のような非ゼロの値に変更します。
- これだけでエラーは解消し、プロファイルが毎日更新されるようになります。

   - AIが正しい「今日」を認識できるように、JST変換を徹底します。
2. **【確実化】二重送信防止ロックの実装**
   - アプリ側で物理的に2回目を防ぐ仕組みを入れます。
3. **【解決】プロファイラーの修正**
   - Pineconeのエラーを回避する修正を入れます。
